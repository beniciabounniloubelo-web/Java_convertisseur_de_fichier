//table canonique

//format PIF
-l'en-tête
taille totale: 4 octets
dont 2 premiers = largeur de l'image
dont 2 derniers = hauteur de l'image

-la section des tables - codes canoniques
taille totale: 768 octets
1) table rouge
2) table vert
3) table bleu
taille d'1 table: 256 octets
1 octet = longueur d'une valeur ; ex: 
Octet 1 → longueur du code pour la valeur 0
Octet 2 → longueur du code pour la valeur 1
…
Octet 256 → longueur du code pour la valeur 255
!!! - si octet avec 0 = valeur pas presente = pas de code

-la section des pixels - codes canoniques
1) Tu lis le code canonique pour R, puis pour G, puis pour B.
2) Tu décodes chaque code canonique pour retrouver la valeur numérique de chaque composante (souvent entre 0 et 255).
3) Ensuite tu peux former le triplet RGB : (R, G, B).
